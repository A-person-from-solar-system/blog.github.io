切面编程

【1】切面编程主要使用【切点】来【拦截】满足条件的方法，使用【通知】在满足条件的方法之前中间或之后执行一些操作，这样能够将方法中【相同的操作】抽离出来。

【2】接下来，通过两个案例来了解切面编程的使用。其中，【案例1】所拦截的是无参的方法，【案例2】所拦截的是有参的方法。

2）准备切点和通知

创建一个类，在该类中添加【切点】和【通知】，通知与切点之间有一定的绑定关系，需要使用注解【@Before】等等，来表明当前通知在所拦截的方法前面还是中间还是后面执行。此外，还需要在注解中指明切点，确定是对那个切点拦截的方法执行当前通知。

package com.itheima.aop;

import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.springframework.stereotype.Component;

@Component
@Aspect  // 注解，表明当前是个切面
public class MyAdvice {
    // 切点
    // 指明拦截的方法为BookServiceImpl的testPoint2()方法
    @Pointcut("execution(void com.coda.service.impl.BookServiceImpl.testPoint2())")
    public void pointCut() {}
	
    // 通知
    // pointCut()表明是对上面的切点所拦截的方法执行下面的通知
    @Before("pointCut()")  // @Before注解是前置通知
    public void method(){
        System.out.println("当前切点对应的【前置通知】已经执行"); 
    }
}

当切面拦截的方法有参，我们可以获取到拦截的方法中的参数，并做一定的修改

【3】使用【args = joinPoint.getArgs()】方法获取拦截到的参数

package com.coda.aop;
import org.aspectj.lang.JoinPoint;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.springframework.stereotype.Component;
import java.lang.reflect.Method;
@Component
@Aspect // 注解，表明当前是个切面
public class MyAdvice {
    // 切点
    // 指明拦截的方法为BookServiceImpl的testPoint2()方法
    @Pointcut("execution(void com.coda.service.impl.BookServiceImpl.testPoint3(*))")
    public void pointCut() {}

    @Before("pointCut()")
    public void method(JoinPoint joinPoint){
        System.out.println("开始通知>>>>>>");
        
        Object[] args = joinPoint.getArgs();  // 获取被拦截方法的参数：Book对象
        if(args==null || args.length==0) {
            return;
        }
        Object entry=args[0];  // Book,默认将传递过来的实体参数放在第一个位置
        String description="作者：兰根";  // 设置需要修改的值

        try{
            // 获得Book类中对应的setDescription方法
            // 需要指明方法名和传入参数类型
            Method setDescription=entry.getClass().getDeclaredMethod("setDescription", String.class);
            setDescription.invoke(entry,description);  // 给Book对象对应的成员赋值
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
        System.out.println("结束通知>>>>>>");
    }
}

@Test
void contextLoads() {
    System.out.println("开始测试：");

    // 准备一个参数
    Book book = new Book();
    book.setId(100);
    book.setType("科普");
    book.setName("重启吧！我的健康人生");
    book.setDescription("作者：某某某"); // 我们故意不给出作者的信息

    // 传递参数
    // 我们使用的是前置通知【在这里修改作者的描述信息】
    bookService.testPoint3(book);  // 调用testPoint3方法【会被切面拦截】
    // testPoint3方法中打印出来的book信息一定是含有作者信息的
    System.out.println("测试结束!");
}

4）结果分析

原先我们输入的作者描述为：【作者：某某某】，然后将book对象传递给testPoint3方法，testPoint3方法复制打印book的值，因为给testPoint3方法添加了【前置通知】，先执行【前置通知】，执行前置通知会修改方法接受的形参内容，将【作者：某某某】修改为【作者：兰根】
